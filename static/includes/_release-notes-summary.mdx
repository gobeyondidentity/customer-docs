### Cloud

                        <span className="changelog-tag" data-tag="New">New!</span>
                    </div>
                    <div class="card-body">
                        <ul>
                            <li class="no-style"><a href="/v2-79-0#bulk-revoke-passkeys"><b>Bulk Revoke Passkeys</b></a>. You can now revoke multiple passkeys directly from the passkey table. </li>
                            <li class="no-style"><a href="/v2-79-0#roaming-authentication-scope"><b>Roaming Authentication Scope</b></a>. You can now scope Roaming Authentication to enable configured source IP addresses. </li>
                            <li class="no-style"><a href="/v2-79-0#policy-attribute-applications-installed"><b>Policy Attribute: Applications Installed</b></a>. The Applications Installed attribute now supports matching on application versions for Windows and macOS.</li>
                            <li class="no-style"><a href="/v2-79-0#policy-action-allow-wbiometric-verification"><b>Policy Action: Allow w/Biometric Verification</b></a>. Beyond Identity administrators can now enforce users to leverage biometrics as a step up/additional factor. </li>
                            <li class="no-style"><a href="/v2-79-0#datadog-siem-integration-enhancement"><b>DataDog SIEM Integration Enhancement</b></a>. Because Datadog offers independent sites based on data governance regulations, you must select the proper site parameter to properly connect to our event HTTP listener.</li>
                            </ul>
                    </div>
                </div>
                
            </div>
        </div>
    </div>
</div>
