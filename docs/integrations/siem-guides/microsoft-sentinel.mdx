## Overview

This guide provides instructions on how to integrate Beyond Identity events data with Microsoft Sentinel. Microsoft Sentinel Integration is done by posting event logs to Entra Log Analytics. The Log Analytics workspace is attached to Microsoft Sentinel.

## Prerequisites

Confirm you have the following prerequisites before continuing with the integration.

- Beyond Identity
    - Tenant configured for organization with ability to enroll users
- Microsoft Sentinel
    - Admin privileges to create Log Analytics workspace
    - Microsoft Sentinel license

## Stage I

The sections in this stage are for the intial setup of the integration between the two platforms.

### ðŸŸ  Configure the Entra Log Analytics workspace

This section has the steps for covering the Microsoft Sentinel setup in your Azure portal.

1. Navigate to your Microsoft Azure portal.
2. Input `log analytics workspaces` into the search box to select the **Log Analytics workspaces**.
3. On the **Log Analytics workspaces** screen, click **Create log analytics workspace**.
4. Configure the following information for the **Create Log Analytics workspace** screen.
    - Select an Entra subscription and resource group
    - Under the **Instance details** section, input `bi-siem-events` as the Name field
    - Select a region for the log analytics workspace
    - Click on **Review + Create**.
5. Click **Create** on the next screen. After the Log Analytics workspace is successfully created, it appears in the **Log Analytics workspaces**.
6. Save the **Workspace Name** and **Workspace ID** to secure space. These values are required for the [Configure Microsoft Sentinel integration in Beyond Identity](#-configure-microsoft-sentinel-integration-in-beyond-identity) section.
7. Click **Agents** and expand **Log analytics agent instructions**.
8. Save the Primary Key value to a secure space. This value is required for the [Configure Microsoft Sentinel integration in Beyond Identity](#-configure-microsoft-sentinel-integration-in-beyond-identity) section.


### ðŸŸ  Add Microsoft Sentinel to the Log Analytics workspace

These steps are for the log analytics workspace for adding Microsoft Sentinel to complete the integration.

1. In the Entra portal, access Microsoft Sentinel.
2. Click **Create**.
3. Under **Add Microsoft Sentinel to a workspace** section, input the `bi-siem-events` as the Workspace name and select the result.
4. Click **Add**.

### ðŸ”µ Add integration in Beyond Identity

The section here includes the steps for configuring the SIEM integration in Beyond Identity. The instructions below are for the Beyond Identity Console.

1. On the Beyond Identity Console lefthand navigation, under Platform, select **Integrations**.
2. Select the **Browse integrations** tab.
3. Select **Microsoft Sentinel**.
4. On the Microsoft Sentinel overview page, click **Add instance**.

### ðŸŸ  Verify logs in Microsoft Sentinel

After completing the integration, view the logs from Beyond Identity within your custom webhook. Follow the instructions below to verify and confirm the data.

1. Navigate to your Microsoft Sentinel workspace on the Entra portal.
2. Select the `bi-siem-events` in the Log Analytics workspace from SECTION.
3. Close the **Queries** pop up.
4. Input the following query in the query text field. Replace `BI_Tenant_events` with the log type value from the Admin console.
```
BI_Tenant_events_CL | where actor_tenant_id_s contains_cs ""
```
5. View the results in the results box. The query language used is [Kusto query language (KQL)](https://learn.microsoft.com/en-us/kusto/query/?view=azure-data-explorer&preserve-view=true)
6. Confirm Beyond Identity events data is available.
