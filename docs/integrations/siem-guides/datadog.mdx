## Overview

This is an instructional guide for integrating Beyond Identity events data with DataDog.

## Prerequisites

Confirm you have the following prerequisites before continuing with the integration.

- Beyond Identity
    - Tenant configured for organization with ability to enroll users
- DataDog
    - DataDog account
    - DataDog API key

## Stage summary

The integration setup consists of the two stages below. Each stage covers the initial set up for continuing the integration process on each respective platform. Follow the guide in linear order to complete the integration.

| Stage | Summary | DataDog | Beyond Identity |
| :---- | :---- | :---- | :---- |
| [I](#stage-i) | Initial configuration | - [Set up DataDog account](#-set-up-datadog-account)<br></br>- [Create DataDog API key](#-create-datadog-api-key) | - [Add integration in Beyond Identity](#-add-integration-in-beyond-identity) |

## Stage I

The sections in this stage are for the intial setup of the integration between the two platforms.

### ðŸŸ  Set up DataDog account

This section has the steps for covering the setup for your DataDog account. Follow the link to log in or create a new account for [DataDog](https://app.datadoghq.com/account/login).

### ðŸŸ  Create DataDog API key

The instructions for this section are for creating an API key in DataDog. With the API key, Beyond Identity posts tenant events into the DataDog log aggregator. To use the Beyond Identity SIEM integration, createa a new API key using the steps below.

1. Copy the DataDog API endpoint below for Beyond Identity tenant events.<br></br>`https://http-intake.logs.datadoghq.com/api/v2/logs`
2. Navigate to your DataDog Account Profile to select **Organization Settings**.
3. Select **API Keys**.
4. Add a new key with the name `Beyond Identity SIEM`.
5. Copy the API key value for the Beyond Identity Console configuration.
6. Save this key value in a secure location.


### ðŸ”µ Add integration in Beyond Identity

The section here includes the steps for configuring the SIEM integration in Beyond Identity. The instructions below are for the Beyond Identity Console.

1. On the Beyond Identity Console lefthand navigation, under Platform, select **Integrations**.
2. Select the **Browse integrations** tab.
3. Select **DataDog SIEM**.
4. On the DataDog overview page, click **Add instance**.

## Stage II

The sections in this stage are for setting up the API key in Beyond Identity and viewing the logs in DataDog. Follow the sections below to complete the integration.

### ðŸ”µ Configure DataDog API in Beyond Identity

The steps below are for adding values from DataDog to Beyond Identity for completing the SIEM integration.

1. On the **Add integration** pop up after completing the steps in [Add integration in Beyond Identity](#-add-integration-in-beyond-identity) section, input the following required values.
    - **Name** â†’ `Beyond Identity SIEM integration`
    - **DataDog Site Parameter** â†’ DataDog API endpoint value from [Step 1 of Create DataDog API key](#-create-datadog-api-key) section
2. Input the API value from [Step 5 of the Create DataDog API key](#-create-datadog-api-key) section into the **API Key** field.
3. For your configuration's events and threat signal selection, click the **Select all and automatically add...** checkboxes for both fields.
    - NOTE: These configuration selections can be changed and updated later.
4. Confirm the **Status** is set to **Active**.
5. Click **Test Configuration**.
6. After testing is complete, click **Save changes**.

### ðŸŸ  Verify logs in DataDog

After completing the integration, view the logs from Beyond Identity within your DataDog account. Follow the instructions below to verify and confirm the data.

1. Navigate to your DataDog URL.
2. On the left pane, select **Logs**.
3. Search using the `source=beyond-identity` pattern to find all Beyond Identity logs within DataDog.
